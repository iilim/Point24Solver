# Point24Solver — 24点算法求解器与挑战平台
---
> 本项目为北京大学 2026 年数据结构与算法（B）课程的课程作业。Point24 Solver 是一个使用 Python 开发的 24 点算法求解器与挑战平台。
---

## 1. 项目背景

**24点**是一款经典的游戏：给定 4 个数字（通常为 1-13 对应扑克牌 A-K），使用四则运算和括号，组合出一个结果等于 24 的表达式。

**创新：本项目在传统功能之外，额外拓展了以用户指定牌数与大小，求解用户指定值的功能。**

本项目不仅实现了基础的求解与交互，更进一步解决了算法工程化中的一些问题：
- **核心求解器**：基于回溯法穷举所有可能解，并去除交换律与结合律产生的等价重复解；
- **安全求值器**：放弃高风险的 `eval()`，利用 Python `ast` 模块构建抽象语法树进行安全解析与校验；
- **CLI 交互模式**：支持正则解析的 `/set` 命令，可动态修改卡牌数量、最大值与目标值（如凑 48 点）；
- **Tkinter 现代图形界面**：扁平化设计，扑克牌风格可视化。

## 2. 核心算法

### 回溯法求解

对于待求解的列表，若其长度为 1，则其首个元素的值为待求目标即收集解；否则，在列表中任选两数进行四则运算，用结果替换原数，得到新列表，进而递归求解。
对于 4 个数字，实测耗时 < 10ms。通过浮点数容差 (`1e-6`) 判断结果，兼顾了性能与精度。

### 去重机制

24点算法常见的痛点是会产生大量数学等价但形式不同的冗余解（如 `4*4+4+4` 和 `4+4+4*4`）。本项目创新性地引入了去重机制：
- 在递归时同步构建规范化的元组结构（如 `('+', (('n', '4'), ('n', '4')))`）。
- 对于加法和乘法，将所有操作数“展平”并按字典序排序生成唯一签名。
- 只要两个表达式数学等价，无论括号位置如何，签名绝对相同，从而在最后一步精准去重，输出最简洁的解。

### 智能括号生成

回溯过程中通过记录每个子表达式的根操作符 (`root_op`)，根据四则运算的优先级与结合律智能决定是否加括号：
- 子表达式优先级低，当前运算符优先级高：必加括号（如 `(4+4)*4`）。
- 优先级相同且为左侧表达式：不加括号（如 `4+4+4`）。
- 优先级相同且为右侧表达式，仅 `-` 和 `/` 加括号（如 `4-(2-1)`）。

### 表达式树

互动模式下，用户输入表达式（如 `(3+5)*(10-7)`）。为了防止恶意代码执行，项目利用 Python 的 `ast` 模块构建表达式树：
1. **语法树遍历**：递归遍历 AST 节点，仅允许二元运算符 (`BinOp`)、一元负号 (`UnaryOp`) 和数字常量 (`Constant`)。
2. **防作弊校验**：在遍历时收集所有叶子节点的数字，与系统发出的卡牌进行比对，确保用户未篡改数字或引入外部变量。

## 4. 运行指南

### 环境要求

- Python 3.8+
- **无需安装任何第三方依赖**（仅使用标准库）

### 安装与运行

```bash
# 克隆仓库
git clone https://github.com/iilim/Point24Solver.git
cd Point24Solver

# 运行 CLI 模式
python src/main.py

# 运行 GUI 图形界面
python src/gui.py
```

### CLI 互动与拓展模式说明

进入 CLI 后选择模式 2 (互动模式)，默认为 4 张牌 (1-13) 凑 24 点。您可以通过正则解析的 `/set` 命令动态修改规则：

```text
【第 1 关】 卡牌: [4, 7, 8, 8]
📌 当前设置: 凑 24 点 | 4 张牌 | 最大值 13
当前得分: 0 | 历史最高: 0
请输入表达式: 7*4-8/8
✅ 回答正确！

# 输入拓展模式指令 (兼容各种空格)
请输入表达式: /set count = 5, target = 48
✅ 设置已更新
📌 当前设置: 凑 48 点 | 5 张牌 | 最大值 13
```
